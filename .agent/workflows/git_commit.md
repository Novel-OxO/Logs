# Git Commit Workflow

이 워크플로우는 변경 사항을 확인하고, 표준화된 메시지 형식으로 커밋 후 원격 저장소에 푸시합니다.

## 커밋 메시지 규칙

커밋 메시지는 **한국어**로 작성하며 다음 형식을 따르는 것을 권장합니다:

`<type>: <description>`

### 타입 (Type)
- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `refactor`: 코드 리팩토링 (기능 변경 없음)
- `docs`: 문서 수정
- `style`: 포맷팅, 세미콜론 누락 등 코드 스타일 수정
- `test`: 테스트 코드 추가/수정
- `chore`: 빌드 업무 수정, 패키지 매니저 설정 등

**예시:** `feat: 로그인 페이지 UI 구현`

## 단계별 실행

### 1. 변경 사항 확인

현재 변경된 파일들을 확인합니다.

// turbo
```bash
git status
```

### 2. 파일 스테이징 (git add)

모든 변경 사항을 스테이징합니다. (특정 파일만 원할 경우 직접 명령어를 수정하세요)

```bash
git add .
```

### 3. 커밋 생성 (git commit)

위의 규칙을 참고하여 커밋 메시지를 작성하고 명령어를 실행하세요.

```bash
git commit -m "<type>: <description>"
```

**예시:**
```bash
git commit -m "docs: 리드미 파일 업데이트"
```

### 4. 원격 저장소 푸시 (git push)

현재 브랜치의 변경 사항을 원격 저장소로 푸시합니다.

```bash
git push origin HEAD
```
